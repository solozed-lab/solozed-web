最近在重构我的个人网站时，我决定采用 Astro v6 搭配 React 19。这个组合带来了一些有趣的思考。

## 为什么选择 Astro？

Astro 的群岛架构让我重新思考网页的性能：

- **零 JavaScript 默认**：只在需要交互的地方加载 JS
- **内容优先**：Markdown/MDX 原生支持
- **框架无关**：可以混用 React、Vue、Svelte

```astro
---
// MyComponent.astro
const { title } = Astro.props;
---

<h1>{title}</h1>
<div class="content">
  <slot /> {/* 内容插槽 */}
</div>
```

## React 19 的新特性

React 19 带来了几个值得关注的变化：

1. **Actions**：表单处理变得更简单
2. **use() Hook**：可以在渲染中读取 promise
3. **改进的 SSR**：更快的首屏渲染

## 两者的结合

关键在于**按需使用**：

| 场景 | 方案 | 原因 |
|------|------|------|
| 博客文章 | Astro | 纯静态，无需交互 |
| 评论系统 | React | 需要实时更新 |
| 背景动画 | React | 需要 canvas 交互 |

```tsx
// OrbBackground.tsx
export function OrbBackground() {
  const canvasRef = useRef<HTMLCanvasElement>(null);

  useEffect(() => {
    // Canvas 动画逻辑
  }, []);

  return <canvas ref={canvasRef} />;
}
```

## 性能对比

使用 Lighthouse 测量：

- **之前**：Performance 78
- **之后**：Performance 96
- **LCP**：从 2.8s 降到 0.8s

## 结论

Astro + React 的组合让我意识到：**没有最好的框架，只有最适合的工具**。静态内容用 Astro，交互组件用 React，各司其职。

对于个人网站来说，这种方式既保证了快速的加载速度，又不失交互的灵活性。
