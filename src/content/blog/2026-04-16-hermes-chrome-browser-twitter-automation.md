我受够了现在网络上的内容。

今天这个话题爆了，一夜之间所有人都在写。明天换个词，同一拨人再写一遍。后天又爆一个，然后继续。

刷抖音的时候如此，刷 X 的时候也如此，满屏都是这种内容。真正有价值的、有观点的、能让我停下来想一想的，反而被淹没在这些重复噪音里。

结果就是——**我每天花大量时间去刷，但真正留下来的信息少得可怜。**

效率极低。但又不得不刷，因为怕错过真正有用的东西。

直到我决定让 AI 替我干这件事。

---

### 先说数据，再说感受

第一件事，先把数字摆出来。

过去我每天刷 X 的时间——我自己估算，大概在 1～2 小时之间。看起来不多，但乘以 7 天，就是将近 7 到 14 个小时。

一个月下来，二三十个小时，就浪费了，这还只是一个平台。

这些浪费的时间，我干了什么？

**划走了。**

没有任何积累。热点过了就过了，大 V 吵完就散了，偶尔收藏了几篇文章，后来再也没有打开过。

这难道就无解吗？

当然不是！

用了 Hermes + Chrome 自动化之后，我改变了信息获取的方式：

不再是「泡在里面找信息」，而是**让 AI 帮我把信息捞出来，我只看它整理好的结果**。

用下来，同样的信息量，我每天花在看 X 上的时间——不超过 15 分钟。

多出来的那些时间，我去哪了？

> 读书、想事情、以及，**认真回复我觉得真正值得讨论的内容。**

总之，可以把更多的时间用在更有意义的事情上。

---

### 第一个没想到：一周多磨合下来，Hermes 越来越懂我了

我原本以为，让 AI 刷 X，抓到的内容肯定很「平均」——它不知道我的偏好，不知道我的关注领域，给我的都是泛泛而谈的热门内容。

**错了。**

一周多磨合下来，我发现它越来越懂我了——不是我手动「训练」了什么，而是我在日常对话里告诉它的偏好，它都会记住，并且在之后的筛选中自动执行。

比如我跟它说过：「我主要看 AI 工具和量化投资相关的内容，娱乐八卦一律不看。」之后它就会自动过滤掉那些账号的日常碎碎念。

它帮我过滤掉的是什么：情绪帖、热点争吵、转发抽奖、以及绝大多数「我关注了但其实并不关心」的内容。

它保留的是什么：**真正有信息量的长文、新的工具或工作流分享、以及值得认真读一遍的独立思考。**

一周下来，我发现一个有意思的现象——

**我获取的信息质量反而变高了，而不是变低了。**

不是 AI 比我会筛选，是它不会被情绪带着走。

它没有「手滑点进去看了半小时」这回事。

---

### 第二个没想到：我重新理解了「信息摄入」这件事

以前我觉得，我刷 X 是因为**怕错过什么**。

怕错过热点，怕错过工具，怕错过某个重要观点，怕自己跟不上。

**这种焦虑感，是刷 X 最大的陷阱。**

你以为是主动获取信息，其实是被信息追着跑。

让 AI 帮我刷了一周之后，我意识到一件事——

**我真正怕的不是错过，是「不知道有这个东西存在」。**

但这两件事根本不是一回事。

AI 帮我筛选之后，我确实错过了一些东西——比如某个热点话题的争吵、某个圈内的八卦、某条刷屏的梗。

**但我真正关心的内容，一篇没漏。**

反而因为 AI 的整理，我对这一周真正重要的事情有了更清晰的认知——不是碎片拼凑出来的模糊印象，而是一份有结构的周报。

---

### 第三个没想到：这让我更想自己动手写了

说来奇怪，AI 帮我减少了信息摄入，但我反而更想自己产出内容了。

可能是因为：

- 每天被高质量内容轰炸，反而激发了「就这？我能写得更好」的冲动
- 节省下来的时间，让我有余力去深度思考，而不是永远在「摄入-焦虑-再摄入」的循环里打转
- AI 帮我筛选信息的这个动作本身，让我更清楚自己**真正关心什么**

这是一个意外的收获。

### 怎么做？避坑先行

说了这么多，你可能会想：「听起来不错，但操作起来麻烦吗？」

**会者不难。**

核心原则只有一件事：**给你的 AI 建一个独立的 Chrome 配置文件。**

这一步你不能省。你的主 Chrome 里有什么？X 账号、知乎、各种登录状态。**你绝对不会想让 AI 在你的主环境里操作。**

独立 Profile = 客房。你让它住进去，它在里面干什么你都知道，和你的主卧完全隔离。

Terminal 里，两条命令搞定：

```bash
mkdir -p ~/Chrome-Hermes-Profile
```

```bash
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
  --user-data-dir="$HOME/Chrome-Hermes-Profile" \
  --remote-debugging-port=9222 \
  --no-first-run &
```

这个路径依照你个人的喜好可以进行修改，但别忘了把下面的路径改成你自己的。

在新打开的 Chrome 窗口地址栏输入 `chrome://version/`，确认配置文件路径指向 `~/Chrome-Hermes-Profile`，然后在这个窗口里登录你的 X 账号。

**日常 Chrome 完全不受影响，两个窗口可以同时开着。**

### 两种方式，选你适合的

> **如果你还不知道 Hermes 是什么？**
>
> 它是一个能 7×24 小时替你干活的 AI 助手框架，支持浏览器自动化、持久记忆、MCP 协议扩展。不是聊天机器人，是数字员工。具体可以看我的另一篇文章《[弃虾投马：我放弃了养虾，加入 Hermes Agent](https://solozed.com/blog/2026-04-08-hermes-agent-openclaw-migration-guide/)》。

**方式一：Hermes 原生连接**

一条命令：

```
/browser connect
```

连上之后，你说什么它做什么——「帮我打开 X，搜 AI 工具，筛选最近 7 天转发超过 50 的帖子，把前 5 条整理给我」。

它像真人一样操作，你等着看结果。

适合：复杂任务、需要 AI 自己判断决策的情况。

**方式二：AutoCLI**

[AutoCLI](https://github.com/nashsu/AutoCLI) 是 Rust 写的极速 CLI 工具，把 55+ 个常用网站变成命令行接口。复用 Chrome 已登录状态，无需任何凭证，单文件 4.7MB，零运行时依赖。

安装 AutoCLI：

```bash
curl -fsSL https://raw.githubusercontent.com/nashsu/autocli/main/scripts/install.sh | sh
```

验证安装：`autocli --version`

Chrome 扩展安装（Browser 模式需要）：

1. 去 [GitHub Releases](https://github.com/nashsu/autocli/releases/latest) 下载 `autocli-chrome-extension.zip`，解压到任意目录
2. Chrome 里打开 `chrome://extensions`
3. 开启右上角「开发者模式」
4. 点击「加载已解压的扩展程序」，选择刚才解压的文件夹

装完扩展会自动连上 AutoCLI daemon。

适合：脚本化、批量抓取、定时任务。

**两者怎么选？**

简单说：AutoCLI 是自动挡，Hermes 是手动挡。日常用 AutoCLI，复杂情况 Hermes 兜底。

这也是我自己现在用的一套模式。

**如果遇到问题，或者疑问，可以在评论区问。**

我更推荐的方法是，自己尝试，遇到问题找 AI 解决，这也是 AI 时代每个人都应该掌握的一项技能。

### 回到那三件事

实验做完，数据摆在眼前，我重新想了一遍这件事的意义。

**第一，信息焦虑的解法不是戒掉 X，是改变获取方式。**

你不需要逃离那个平台，你需要的是不再被算法和情绪牵着走。

**第二，AI 不是来取代你的判断的，是来放大你的判断力的。**

它帮你筛，你来做决定。哪些值得深读，哪些值得回应，哪些值得分享——这还是你的事。

**第三，省下来的时间，是用来真正思考的。**

不是用来看更多的信息、刷更多的帖子。是安静地想一件事，然后把它写出来。

这才是信息摄入应有的样子。
