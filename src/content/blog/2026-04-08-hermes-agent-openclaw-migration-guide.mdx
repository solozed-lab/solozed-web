## 放弃养虾，加入 Hermes（爱马仕）Agent

 **Hermes Agent** 这个项目，起初只是觉得名字有点意思，随手试了一下，没想到这一试，直接让我决定从 OpenClaw 叛逃了。

用圈子里的话说，我开始弃虾投马。

## Hermes Agent 是什么

[![Star History Chart](https://api.star-history.com/chart?repos=nousresearch/hermes-agent&type=date&logscale&legend=top-left)](https://www.star-history.com/?repos=nousresearch%2Fhermes-agent&type=date&logscale=&legend=top-left)

简单来说，Hermes Agent 是一个 AI 编程助手框架，它能帮你处理各种开发任务——写代码、调试、重构、搜索代码库，甚至帮你做技术决策。

**核心特征：**

- 多平台通信：通过统一网关连接 14+ 消息平台
- 持久记忆：跨会话记忆系统，不会“失忆”
- 自主学习：能从经验中生成可复用的技能
- 灵活部署：本地、Docker、SSH、云端均可运行
- 47 个内置工具：文件操作、网页搜索、浏览器自动化、语音交互等
- MCP 协议兼容：可对接外部工具服务器，扩展能力无上限

它的定位不是**一个聊天机器人**，而是一个能 **7×24** 小时替你干活的数字员工。

但如果只是这样，跟 OpenClaw 有什么区别？

区别在于：Hermes 更懂你的项目。

它不是那种你问什么它答什么的工具，而是能真正理解项目结构、记住你的偏好、在多个会话间保持上下文一致的助手。有点像是给你的开发环境装上了一个一直在旁边观察学习的老员工——不用你反复解释背景，它自己就能跟上节奏。

## 为什么我从 OpenClaw 转过来了

说实话，OpenClaw 陪伴我很长一段时间。它的提示词优化、代码审查、多 agent 协作模式都很成熟。但用久了，有几个痛点开始让我不舒服：

**1. 上下文丢失太频繁**

每次开新会话，OpenClaw 需要我重新描述项目背景。做一些大型重构的时候，割裂感特别明显。Hermes 的多会话记忆系统让我不再需要重复劳动。

**2. 工具链的颗粒度**

Hermes 的工具集更细。比如它的 `browser_navigate` 可以直接操作浏览器，`execute_code` 支持多语言沙盒执行，而 `delegate_tool` 可以让你把任务分配给子 agent 处理——这些组合起来，能应对更复杂的场景。

**3. 配置灵活性**

Hermes 支持 profiles（多实例隔离），我可以在不同项目间切换完全独立的配置、记忆和 API 设置。这对同时维护多个项目的人来说简直是救星。

## 核心功能一览

### 智能代码搜索与导航

Hermes 的搜索不是简单的关键词匹配。它能理解代码语义，找函数调用链、查变量定义、分析依赖关系。比如你想知道某个 API 在某处被调用过，直接问就行，不用再 grep 来 grep 去。

### 会话记忆与上下文保持

这是最让我惊喜的功能。Hermes 会自动保存会话轨迹，关键决策和技术选型的讨论都会被记录。下次遇到类似问题，它能调取"记忆"给你参考。

### 工具生态

目前 Hermes 支持的工具覆盖：

- **浏览器操作** — 自动化的网页交互、表单填写、截图
- **终端执行** — 直接在命令行跑脚本、跑测试
- **文件处理** — 读写、搜索、批量修改
- **代码执行** — 多语言沙盒，即时验证代码片段
- **子 agent 委托** — 把复杂任务拆分给专门的 agent 处理

### Slash 命令系统

用 `/` 开头的命令直接触发各种功能——`/plan` 做技术规划、`/review` 做代码审查、`/model` 切换模型……输入效率大幅提升。

## 从 OpenClaw 迁移？几点建议

如果你决定试试 Hermes，有几个小建议：

**从 CLI 开始** — 先在日常小任务上体验，比如让它帮你查一段代码、解释一个 error、润色一段注释。别一上来就扔给它大型重构。

**保留 OpenClaw 作为对比** — 不是说非此即彼。两个工具可以并存，看哪个场景下谁更好用。

**关注 `~/.hermes/config.yaml`** — 这是 Hermes 的配置中心。好好研究里面的选项，特别是 `display`、`tools`、`models` 这几个部分。

**用好 session 机制** — 同一个项目的多次对话尽量保持 session 连续。这样 Hermes 能积累足够的上下文，回复质量会明显提升。

## 什么场景下 Hermes 特别强

根据我的体验，这几类任务 Hermes 表现尤其好：

- **大型项目的重构** — 它能理解全代码库结构，跨文件修改更精准
- **技术调研** — 帮你查文档、搜方案、整理成可操作的建议
- **代码审查** — 不仅找 bug，还能指出设计层面的问题
- **自动化脚本** — 直接生成、测试、调试一条龙
- **多人项目的知识传承** — 新成员加入时，可以用 Hermes 快速了解项目历史和技术决策

## 适合谁用？

- 独立开发者：让 Agent 在 VPS 上 24 小时待命，随时通过 Telegram 下达任务
- 小团队：通过 Slack/Discord 共享一个 Agent，处理运维、监控、自动化任务
- AI 研究者：接入本地模型，在 HPC 集群上跑实验
- 自动化爱好者：用 cronjob 定时执行任务，用消息网关接收结果通知
- 重视隐私的用户：完全本地部署，数据不离开你的服务器

## 当然，它也有不足

我不想把 Hermes 吹上天。有些地方它还需要改进：

- 首次配置还是有点门槛，对新手不够友好
- 部分工具需要额外的 API Key（比如某些网页自动化功能）
- 文档还在完善中，有些高级用法需要自己摸索

但总体来说，它的进化速度很快。每次更新都能看到明显的改进。

## 最后说一句

工具没有最好，只有最适合。

OpenClaw 教会了我很多，也让我的开发效率提升了一个档次。我不会说它的坏话——它依然是个好工具。

但 Hermes 在上下文理解、多会话记忆和工具灵活性上的优势，让它在处理复杂、长期项目时更加得心应手。

如果你也在找一个新的 AI 编程助手，或者只是想看看另一片天是什么样的，Hermes 值得你花一个下午去体验。
